What you actually give
the coding agent.

The architecture notes are reference material, not prompt fodder. This doc is the operational bundle — the CLAUDE.md hierarchy, the repos to clone, the phase-scoped prompts, and the escape hatches for decisions an agent can't make alone.

§ 01

The approach

Agent delegation fails when you paste everything into one megaprompt. It also fails when you paste too little and the agent has to guess. The pattern that actually works is a CLAUDE.md hierarchy — a top-level rules file at the repo root, per-directory CLAUDE.md files for area conventions, and a .agents/ folder for context and skills.

The agent reads the top-level CLAUDE.md on every turn, but only reads subtree CLAUDE.mds when it enters those subtrees. That gives you layered context without overwhelming every conversation. Reference documents live in .agents/context/; the agent pulls them in via cat when relevant.

§ 04

Top-level CLAUDE.md

The root rules file. Read on every turn. Keep it under 200 lines.

CLAUDE.mdcopy verbatim
# T3 Cloud Planner

Cloud planner integration. An async agent that runs as a durable Convex
workflow, reads a synced repo via convex-sandbox + just-bash, produces
structured plans, dispatches executable steps to T3 Code desktop.

## Stack

- Backend: Convex (TypeScript strict)
- Agent: @convex-dev/agent + AI Gateway + bash-tool
- Workflow: @convex-dev/workflow for durability

## Non-negotiable conventions

1. TypeScript strict mode. No any outside storage boundaries.
2. Convex v.* validators on every mutation/query/action.
3. ULIDs for event IDs, assigned at creation.
4. Idempotency keys on every state-changing mutation.

General re-engagement

→ resume work
Check git status and the open PRs. Read CLAUDE.md and the current phase's task file. Tell me the current state and the next two tasks you'd pick up. Don't start coding until I confirm.
§ 06

Skills to pre-create

new-convex-function
Invoked: adding any new mutation/query/action. Enforces: validator shape, auth check, colocated test file.
new-workflow-step
Invoked: extending the orchestrator workflow. Enforces: idempotency, proper step.runAction vs step.runMutation.
new-event-kind
Invoked: adding a new kind to the event log. Enforces: envelope shape, Zod schema, ULID generation.
§ 07

What the agent can't do alone

Decisions the agent should flag and wait on, not guess.

Authorization model for sandbox-to-repo sync.
Flag: affects sandboxes.repoSourceUrl and how pinnedCommit is obtained.
Default model for the planner.
Flag: PRD says claude-sonnet-4 default but it's an open question. Confirm before prompt-tuning locks in.
Usage bar exhaustion mid-run behavior.
Flag: does the run pause? Complete partially? Overage-charge? Don't implement until decided.
Operating principle
The agent is good at mechanical correctness. It's bad at knowing what you want the product to be. Keep mechanical work in the prompt scope; keep product decisions in the escape hatches.
